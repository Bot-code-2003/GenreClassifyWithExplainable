Genre Classifier

A machine learning model that classifies textual summaries into literary genres like Fantasy, Crime, Romance, Horror, and more.

Overview

The Genre Classifier is a machine learning-based application designed to identify the literary genre of a given text input such as a story summary, a plot description, or a short paragraph. It supports 10 popular genres including Fantasy, Horror, Romance, Science Fiction, Crime, and more.

The system is powered by a trained ensemble model combining logistic regression, random forests, and gradient boosting techniques. Text inputs undergo TF-IDF vectorization and are balanced using SMOTE to address class imbalances in the training dataset.

Explainable AI

A major feature of this application is its integration of LIME (Local Interpretable Model-agnostic Explanations), which allows users to visualize which words influenced the prediction and to what degree. This enhances transparency and trust in model predictions.

The frontend interface provides an intuitive user experience, with confidence scores and an interactive bar chart built using Chart.js. For deeper insight, users can generate detailed explanations that reveal how the model interprets different genre possibilities.

Predict Endpoint

POST /predict

Predicts the genre of the submitted text.

Request Body
JSON 
{
  "text": "A group of friends explore an ancient castle filled with dark secrets..."
}
Response 200 OK
JSON 
{
  "predicted_genre": "horror",
  "confidence": 0.72,
  "prediction_probabilities": {
    "fantasy": 0.08,
    "horror": 0.72,
    "crime": 0.12,
    "romance": 0.03,
    "history": 0.05
  }
}

Explain Endpoint

POST /explain

Opens an HTML visual explanation using LIME.

Request Body
JSON 
{
  "text": "A group of friends explore an ancient castle filled with dark secrets..."
}
Response 200 OK

Output: Opens a webpage with:

  • Predicted genre
  • Top contributing words
  • Chart of LIME word importances

Model Pipeline

1. Preprocessing

Input text is normalized through several NLP operations: lowercasing, punctuation removal, stopword removal, and lemmatization using nltk. This ensures consistent representation and reduces noise before vectorization.

2. Feature Extraction

The cleaned text is converted into numerical form using TF-IDF vectorization, capturing both term frequency and inverse document frequency. We use unigrams and bigrams (1-2 word phrases) with a max feature limit of 10,000 to retain semantic richness while avoiding overfitting.

3. Data Balancing

Genre distributions are naturally imbalanced. To mitigate this, we use SMOTE (Synthetic Minority Over-sampling Technique), which generates synthetic examples for underrepresented genres — ensuring the model trains equally across all classes.

4. Model Training

An ensemble voting classifier is trained using three base models: Logistic Regression, Random Forest, and Gradient Boosting. Soft voting is used to average their predicted probabilities, yielding a robust and generalizable final output.

5. Explainability

Post-prediction, we apply LIME (Local Interpretable Model-agnostic Explanations) to understand why a genre was selected. It highlights key words that influenced the model decision, making the black-box prediction process transparent and interpretable for end users.
